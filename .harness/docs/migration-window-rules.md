# 迁移窗口规则

用户关于 **default-soft 迁移 + EOL 物理删除** 的规范。源于真实生产教训——太激进的 breaking change（"步子跨大扯着蛋"）和太软的"永远 deprecate 反正没人翻"开关，中间路线是：发货安全默认值，给一个真实的截止日期，然后物理删除。

## 规则优先级顺序

### 1. 新开关默认 = 安全方向

- 如果变更让功能**默认更安全**，新开关默认 `true`（仅当安全行为是新行为时）
- 如果变更让功能**默认更有风险**（如新的激进缓存驱逐、新的熔断阈值），新开关默认 `false`，需要显式 opt-in
- 默认值必须是对用户（忙碌的 solo 开发者）不读 changelog 也能得到的安全行为

### 2. 每个新开关都需要 `@MigrationWindow`

注解（在 `atlas-richie-component-cache` 或所在位置）记录：
- `owner` — 谁负责删除开关（如 `cache-team` 负责 cache-component 开关）
- `deadline` — 物理删除的日期
- `description` — 开关做什么、用户应该做什么

```java
@MigrationWindow(
    owner = "cache-team",
    deadline = "2026-12-31",
    description = "新激进驱逐策略的临时开关；截止日期删除。用户需显式设置新属性来迁移。"
)
private boolean useAggressiveEviction = false;
```

### 3. 截止日期必须是真实的

- 截止日期超过约 12 个月太软了——用户会忘
- 截止日期少于一个发布周期（约 1 个月）太紧了——用户没有迁移时间
- 默认节奏：**2 个发布周期（约 2 个月）后删除**

### 4. 窗口是契约，不是猜测

- 如果截止日期快到了但迁移还没完成，**可以延期一次**并附上说明。用户被"无限延期"软开关坑过。
- 延期之后，截止日期就是最终的了。不再延期。

### 5. 截止日期后，**物理删除**

- 从代码中移除开关
- 从 `*Properties.java` 中移除属性
- 移除条件逻辑
- 更新 CHANGELOG："已删除 X；用户现在必须做 Y"
- **不要 deprecate。** Deprecate 是给那些永远留下的东西用的。这是一个有删除日期的临时开关。

### 6. 不要引入"没人翻的软开关"

如果一个开关默认 `false` 超过迁移窗口，且**没有业务代码翻过它**，这是信号：
- 要么开关不需要（删除它，不需要迁移）
- 要么业务代码不知道要迁移（通知；但不延期）

用户明确说过："信任人们会响应截止日期"。

### 7. 在同一个 PR 里记录迁移

引入开关的 PR 必须：
- 加 `@MigrationWindow` 注解
- 在相关 README 中更新新属性和截止日期
- 在 CHANGELOG 加"Deprecation" / "Migration" 条目
- 加示例或测试展示两种状态（旧 + 新）

## 用户点名的反模式

> "为业务代码做兜底和下限，防止业务代码无下限" — 迁移窗口是一种兜底形式：它防止半迁移的代码库在下次重构时静默崩溃。

> "Default-soft" 不等于"永远软"。没有截止日期的开关是一个永远不会被翻的软开关。

> "步子跨大扯着蛋" — 不要试图一个 PR 迁移所有人。窗口让你立即发货安全默认值，迁移在用户的节奏上发生。

## 注解在哪里

在以下位置查找：
- `atlas-richie-component-cache/` — 很可能是规范位置（刚在 `@MigrationWindow` 功能工作中加入）
- `atlas-richie-base/atlas-richie-context/` — 可能是共享注解

找不到就查 `master` 最近提交中对 `MigrationWindow` 的引用。

## 验证迁移窗口有效

- [ ] `owner` 设为真实的人 / 团队
- [ ] `deadline` 至少一个发布周期之后、最多约 12 个月
- [ ] 开关默认指向用户的安全行为
- [ ] README + CHANGELOG 提到它
- [ ] 示例 / 测试展示了两种状态

任何一项缺失，PR 就算未完成。

## 不要做的事

- 加了带截止日期的开关但到截止日期忘了删——设提醒
- 延期但不写理由
- deprecate（加 `@Deprecated`）然后留着开关——物理删除或什么都不做
- 当一个普通代码变更（加适当的迁移文档）就能搞定时，引入开关
