# richie-component-tracing 依赖说明

## 📦 新增依赖（2025-10-30）

由于 `richie-component-cache` 中的 OpenTelemetry 依赖已改为可选依赖，`richie-component-tracing` 作为专门的链路追踪组件，需要显式声明所有必需的依赖。

### ✅ 新增的依赖

#### 1. **opentelemetry-exporter-zipkin**
```xml
<dependency>
    <groupId>io.opentelemetry</groupId>
    <artifactId>opentelemetry-exporter-zipkin</artifactId>
</dependency>
```

**作用**：支持将追踪数据导出到 Zipkin
**使用场景**：
- 当用户使用 Zipkin 作为追踪后端时
- 提供除 Jaeger 之外的另一种选择

**配置示例**：
```java
-Dotel.traces.exporter=zipkin
-Dotel.exporter.zipkin.endpoint=http://localhost:9411/api/v2/spans
```

#### 2. **opentelemetry-spring-boot-starter**
```xml
<dependency>
    <groupId>io.opentelemetry.instrumentation</groupId>
    <artifactId>opentelemetry-spring-boot-starter</artifactId>
</dependency>
```

**作用**：Spring Boot 应用的 OpenTelemetry 自动配置
**使用场景**：
- 不使用 JavaAgent 方式
- 想在应用代码中编程式配置 OpenTelemetry
- 需要更精细的控制

**配置示例**：
```yaml
otel:
  sdk:
    disabled: false
  traces:
    exporter: otlp
  exporter:
    otlp:
      endpoint: http://localhost:4317
  service:
    name: my-service
```

#### 3. **micrometer-registry-otlp** (可选)
```xml
<dependency>
    <groupId>io.micrometer</groupId>
    <artifactId>micrometer-registry-otlp</artifactId>
    <optional>true</optional>
</dependency>
```

**作用**：支持将 Micrometer metrics 导出到 OTLP
**使用场景**：
- 需要同时收集 traces 和 metrics
- 使用统一的 OTLP 端点

**配置示例**：
```yaml
management:
  otlp:
    metrics:
      enabled: true
      endpoint: http://localhost:4318/v1/metrics
```

### 集成

- `micrometer-registry-otlp` - Micrometer 集成（可选）✨ **新增**

## 🎯 使用建议

### 场景一：使用 JavaAgent（推荐）

**优势**：
- 无需修改代码
- 自动 instrumentation
- 配置简单

**使用方式**：
```bash
java -javaagent:/path/to/opentelemetry-javaagent.jar \
     -Dotel.traces.exporter=otlp \
     -Dotel.exporter.otlp.endpoint=http://localhost:4317 \
     -Dotel.resource.attributes=service.name=my-service \
     -jar your-app.jar
```

**依赖要求**：
- 不需要在项目中引入 `richie-component-tracing`
- JavaAgent 自包含所有必需的依赖

### 场景二：应用内集成

**优势**：
- 更精细的控制
- 可以编程式配置
- 适合复杂场景

**使用方式**：
```xml
<dependency>
    <groupId>com.richie.component</groupId>
    <artifactId>atlas-richie-component-tracing</artifactId>
</dependency>
```

**配置方式**：
通过 `application.yml` 或编程式配置 OpenTelemetry

### 场景三：同时收集 Traces 和 Metrics

**使用方式**：
```xml
<dependency>
    <groupId>com.richie.component</groupId>
    <artifactId>atlas-richie-component-tracing</artifactId>
</dependency>
```

**配置**：
```yaml
# Traces 配置
otel:
  traces:
    exporter: otlp
  exporter:
    otlp:
      endpoint: http://localhost:4317

# Metrics 配置
management:
  otlp:
    metrics:
      enabled: true
      endpoint: http://localhost:4318/v1/metrics
```

## 🔧 支持的导出器

| 导出器 | 后端系统 | 端口 | 协议 |
|--------|---------|------|------|
| OTLP | OpenTelemetry Collector | 4317 (gRPC), 4318 (HTTP) | gRPC/HTTP |
| Jaeger | Jaeger | 14250 (gRPC), 14268 (HTTP) | gRPC/HTTP |
| Zipkin | Zipkin | 9411 | HTTP |
| Logging | Console/Log File | - | - |

## 🎓 迁移说明

如果您之前依赖 `richie-component-cache` 传递的 OpenTelemetry 依赖，现在需要：

1. **显式引入 tracing 组件**：
   ```xml
   <dependency>
       <groupId>com.richie.component</groupId>
       <artifactId>atlas-richie-component-tracing</artifactId>
   </dependency>
   ```

2. **或者直接引入 OpenTelemetry 依赖**：
   ```xml
   <dependency>
       <groupId>io.opentelemetry.instrumentation</groupId>
       <artifactId>opentelemetry-spring-boot-starter</artifactId>
   </dependency>
   ```

3. **推荐使用 JavaAgent 方式**（无需引入任何依赖）
