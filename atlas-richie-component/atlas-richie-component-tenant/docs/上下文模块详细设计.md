## 上下文模块详细设计

### 1. 模块定位

提供统一的租户 ID 持有与传递机制，对外仅暴露 `TenantContext` 门面，内部根据配置自动在 `ScopedValue` 与 `TransmittableThreadLocal` 之间切换，**无需业务代码或框架显式处理线程间上下文传播**。

### 2. 核心类型

#### 2.1 TenantContextHolder 接口

```java
public interface TenantContextHolder {
    void runWithTenant(TenantPrincipal principal, Runnable task);
    TenantPrincipal get();
    void clear();
}
```

#### 2.2 ScopedValueHolder

```java
public class ScopedValueHolder implements TenantContextHolder {
    private static final ScopedValue<TenantPrincipal> CURRENT_TENANT = ScopedValue.newInstance();

    @Override
    public void runWithTenant(TenantPrincipal principal, Runnable task) {
        ScopedValue.where(CURRENT_TENANT, principal).run(task);
    }

    @Override
    public TenantPrincipal get() {
        return CURRENT_TENANT.isBound() ? CURRENT_TENANT.get() : null;
    }

    @Override
    public void clear() {
        // 自动清除，空实现
    }
}
```

#### 2.3 TransmittableThreadLocalHolder

```java
import com.alibaba.ttl.TransmittableThreadLocal;

public class TransmittableThreadLocalHolder implements TenantContextHolder {
    private static final TransmittableThreadLocal<TenantPrincipal> CURRENT_TENANT = new TransmittableThreadLocal<>();

    @Override
    public void runWithTenant(TenantPrincipal principal, Runnable task) {
        TenantPrincipal previous = CURRENT_TENANT.get();
        CURRENT_TENANT.set(principal);
        try {
            task.run();
        } finally {
            if (previous != null) {
                CURRENT_TENANT.set(previous);
            } else {
                CURRENT_TENANT.remove();
            }
        }
    }

    @Override
    public TenantPrincipal get() {
        return CURRENT_TENANT.get();
    }

    @Override
    public void clear() {
        CURRENT_TENANT.remove();
    }
}
```

### 3. 门面与初始化

```java
if (properties.isForceThreadLocal() || !isScopedValueAvailable()) {
    holder = new TransmittableThreadLocalHolder();
} else {
    holder = new ScopedValueHolder();
}
```

### 4. 租户身份解析

#### 4.1 身份来源

| 来源 | 权威性 | 获取方式 |
|------|--------|---------|
| JWT `tenantCode` claim | **权威源** | `JwtUtils.getTenantCode(token)` |
| `X-Tenant-ID` Header | **辅助校验**，须与 JWT 一致 | `request.getHeader("X-Tenant-ID")` |
| JWT `tenantExpiredTime` claim | 租户账户过期检测 | `JwtUtils.getTenantExpiredTime(token)` |

**安全原则**：`X-Tenant-ID` 为不可信输入，仅用于与 JWT 交叉校验。禁止仅信任 Header 确定租户身份。

#### 4.2 TenantIdentityFilter

在 Filter 链中位于 Spring Security 认证过滤器之后执行，负责从 JWT 提取租户身份并绑定到 `TenantContext`。

```java
/**
 * 租户身份解析过滤器。
 * 从 JWT 提取 tenantCode，与 X-Tenant-ID header 交叉校验后绑定到 TenantContext。
 * 执行时机：Spring Security 认证完成后（FilterOrder: TENANT_IDENTITY = 500）。
 */
@WebFilter(urlPatterns = "/**")
public class TenantIdentityFilter extends OncePerRequestFilter {

    private static final String JWT_CLAIM_TENANT = "tenantCode";
    private static final String JWT_CLAIM_EXPIRED_TIME = "tenantExpiredTime";
    // header key 从配置读取：properties.getTenantIdHeader()，默认 "X-Tenant-ID"

private final MultiTenancyProperties properties;
private final TenantInfoProvider tenantInfoProvider;
private final AntPathMatcher pathMatcher = new AntPathMatcher();

    @Override
    protected boolean shouldNotFilter(HttpServletRequest request) {
        String path = request.getRequestURI();
        // 白名单：公开端点（登录、健康检查、静态资源等）跳过解析
        return pathMatcher.match("/auth/**", path)
            || pathMatcher.match("/actuator/**", path)
            || pathMatcher.match("/public/**", path)
            || pathMatcher.match("/error", path);
    }

    @Override
    protected void doFilterInternal(HttpServletRequest request,
                                     HttpServletResponse response,
                                     FilterChain chain) throws IOException, ServletException {

        // 1. 从 JWT 提取 tenantCode（权威源）
        String token = extractToken(request);
        if (token == null) {
            response.sendError(401, "Missing authentication token");
            return;
        }

        String tenantCode = JwtUtils.getTenantCode(token);

        // 2. 无 tenantCode → 平台超管，不绑定租户上下文
        if (tenantCode == null) {
            chain.doFilter(request, response);
            return;
        }
        if (tenantCode.isBlank()) {
            response.sendError(403, "JWT tenantCode claim is blank");
            return;
        }

        // 3. 租户白名单校验
        if (!tenantCode.matches(properties.getAllowedTenantPattern())) {
            response.sendError(403, "Invalid tenant code format: " + tenantCode);
            return;
        }

        // 4. 租户过期检测
        Long expiredTime = JwtUtils.getTenantExpiredTime(token);
        if (expiredTime != null && System.currentTimeMillis() > expiredTime) {
            response.sendError(403, "Tenant account expired: " + tenantCode);
            return;
        }

        // 5. 与 tenant-id-header 交叉校验（从配置读取，如果 header 存在）
        String headerTenantId = request.getHeader(properties.getTenantIdHeader());
        if (headerTenantId != null && !headerTenantId.equals(tenantCode)) {
            response.sendError(403,
                "Tenant mismatch: JWT=" + tenantCode + " vs Header=" + headerTenantId);
            return;
        }

        // 6. 租户存在性校验（杜绝未知租户攻击）
        if (!tenantInfoProvider.exists(tenantCode)) {
            response.sendError(403,
                "Unknown tenant: " + tenantCode
                + ". Please register via management API first.");
            return;
        }

        // 7. 绑定租户上下文
        TenantPrincipal principal = TenantPrincipal.builder()
            .tenantId(Long.valueOf(tenantCode))
            .expiredTime(expiredTime != null
                ? java.time.Instant.ofEpochMilli(expiredTime).atOffset(java.time.ZoneOffset.UTC)
                : null)
            .build();
        TenantContext.runWithTenant(principal, () -> {
            try {
                chain.doFilter(request, response);
            } catch (IOException | ServletException e) {
                throw new RuntimeException(e);
            }
        });
    }

    private String extractToken(HttpServletRequest request) {
        // 优先从 HeaderContextHolder 获取（已在 Gateway 解析过）
        String token = HeaderContextHolder.getString("X-Access-Token");
        if (token != null) return token;
        // 降级：从 Authorization header 解析 Bearer token
        String auth = request.getHeader("Authorization");
        if (auth != null && auth.startsWith("Bearer ")) {
            return auth.substring(7);
        }
        return null;
    }
}
```

#### 4.3 身份解析错误矩阵

| 场景 | HTTP 状态码 | 说明 |
|------|------------|------|
| 无认证 Token | 401 | 未登录或 Token 已过期 |
| JWT 无 `tenantCode` claim | 跳过 | 平台超管，不绑定租户上下文 |
| `tenantCode` 不符合 `allowed-tenant-pattern` | 403 | 拒绝非法字符注入 |
| 租户在 `sys_tenant` 中不存在 | 403 | 未知租户，引导通过管理接口注册 |
| 租户账户已过期 | 403 | `tenantExpiredTime` < 当前时间 |
| JWT `tenantCode` ≠ `X-Tenant-ID` header | 403 | 防止租户伪造 |
| 白名单路径 | 跳过 | 公开端点不绑定租户上下文 |

#### 4.4 特殊场景

**超级管理员（无租户归属）**：

```java
// JWT 中无 tenantCode claim → 平台超管，不绑定租户上下文
if (tenantCode == null) {
    chain.doFilter(request, response);
    return;
}
```

> 当前 JWT 中无 `isAdmin` 或 `role` claim。判断超管的自然方式就是 `tenantCode` 不存在——`JwtUtils` 已原生支持 `tenantCode` 为 null 的场景（`verify()` 中对 null 做空串兜底）。

**微服务内部调用（Feign）**：通过 `TenantFeignInterceptor` 自动将 `TenantContext` 中的租户 ID 注入到 Feign 请求的 `X-Tenant-ID` header。详见 §7.2 OpenFeign 传递的完整实现。

**MQ / 定时任务**：已在上下文模块 §MQ 场景中说明，生产者主动放入消息头，消费者从消息头提取后调用 `runWithTenant()` 绑定。

#### 4.5 端到端流转

```
Gateway TenantFilter (order=300)
  JwtUtils.getTenantCode(X-Access-Token) → tenantCode
  JwtUtils.getTenantExpiredTime(X-Access-Token) → 过期检测
  ↓ (tenantCode 写入 HeaderContextHolder 透传)
  
↓ 下游微服务...

TenantIdentityFilter (order=500)
  1. 从 HeaderContextHolder / Authorization 提取 JWT
  2. JwtUtils.getTenantCode(token) → tenantCode
  3. JwtUtils.getTenantExpiredTime(token) → 过期检测
  4. X-Tenant-ID header 交叉校验
  5. 构造 TenantPrincipal → TenantContext.runWithTenant(principal, chain.doFilter)
  
↓ Filter 链继续...

TenantStrategyInterceptor (order=拦截器链 #2)
  TenantContext.get() → 获取当前 TenantPrincipal
  TenantInfoProvider.getTenantInfo(principal.getTenantId()) → 隔离策略 + 数据源路由
```

### 5. 线程池与 MQ 上下文传播

`ScopedValue` **不跨线程继承**（JDK 刻意设计——仅在 `StructuredTaskScope` 内虚拟线程间传递）。因此线程池传播策略取决于当前 Holder 模式。

#### 5.1 模式差异

| 特性 | ScopedValue 模式 | ThreadLocal（TTL）模式 |
|------|-----------------|----------------------|
| 跨线程池自动继承 | ❌ 不支持 | ✅ TTL 自动传递 |
| StructuredTaskScope 内虚拟线程 | ✅ 自动继承 | ⚠️ 需 TTL 包装 |
| required approach | 显式捕获 → `runWithTenant()` 恢复 | `TtlExecutors` 包装 或 显式捕获 |

#### 5.2 通用方案：TaskDecorator（推荐，双模式兼容）

对 Spring `@Async` / `TaskExecutor`，使用 `TaskDecorator` 在任务提交时捕获、执行时恢复，对两种模式均生效：

```java
/**
 * 租户上下文 TaskDecorator。
 * 提交异步任务时从当前线程捕获 tenantId，执行时恢复。
 * ScopedValue 与 ThreadLocal 模式均兼容。
 */
public class TenantTaskDecorator implements TaskDecorator {

    @Override
    public Runnable decorate(Runnable runnable) {
        TenantPrincipal principal = TenantContext.get();  // 在主线程捕获
        if (principal == null) {
            return runnable;
        }
        return () -> TenantContext.runWithTenant(principal, runnable);
    }
}

// Spring 配置
@Configuration
public class AsyncConfig implements AsyncConfigurer {
    @Override
    public Executor getAsyncExecutor() {
        ThreadPoolTaskExecutor executor = new ThreadPoolTaskExecutor();
        executor.setTaskDecorator(new TenantTaskDecorator());
        executor.initialize();
        return executor;
    }
}
```

#### 5.3 TTL 包装方案（仅 ThreadLocal 模式）

若确认使用 `force-thread-local: true`，也可沿用 TTL 包装：

```java
Executor originalPool = Executors.newFixedThreadPool(10);
Executor ttlPool = TtlExecutors.getTtlExecutor(originalPool);
ttlPool.execute(() -> {
    // TenantContext.get() 自动可用
});
```

> TTL 包装在 ScopedValue 模式下**无效**——ScopedValue 不会通过 TTL 传播。因此推荐始终使用 §5.2 的 `TaskDecorator` 方案，不依赖底层 Holder 实现。

#### 5.4 MQ

- **生产者**：发送前从 `TenantContext.get()` 获取 `TenantPrincipal`，将其 `tenantId` 放入消息头
- **消费者**：从消息头提取 tenantId，构造 `TenantPrincipal` 后调用 `TenantContext.runWithTenant(principal, () -> { ... })` 绑定后执行业务逻辑

### 6. 与策略层的交互

所有策略（Column、Table 等）直接调用 `TenantContext.get().getTenantId()`，对底层实现无感知。在 Table/Schema/Database 策略中需切换租户时，使用 `runWithTenant` 绑定。

### 7. 跨服务租户传递

当请求跨越微服务边界时，租户 ID 必须随请求一起传递到下游服务，确保全链路租户上下文一致。

**传递协议**：所有出站协议统一通过 `X-Tenant-ID`（由 `multi-tenancy.tenant-id-header` 配置，默认 `X-Tenant-ID`）传递当前 `TenantContext` 中的租户 ID（`TenantContext.get().getTenantId()`）。下游服务入口的 `TenantIdentityFilter` 收到后与 JWT `tenantCode` 交叉校验。

> **安全原则**：`X-Tenant-ID` 是辅助校验字段，不是可信输入。下游服务的 `TenantIdentityFilter` 始终以 JWT `tenantCode` 为权威源，仅用 `X-Tenant-ID` 做一致性校验。

#### 7.1 HTTP 传递

**出站（RestTemplate）**：

```java
public class TenantClientHttpRequestInterceptor implements ClientHttpRequestInterceptor {

    private final MultiTenancyProperties properties;

    public TenantClientHttpRequestInterceptor(MultiTenancyProperties properties) {
        this.properties = properties;
    }

    @Override
    public ClientHttpResponse intercept(HttpRequest request, byte[] body,
                                         ClientHttpRequestExecution execution) throws IOException {
        TenantPrincipal principal = TenantContext.get();
        if (principal != null) {
            request.getHeaders().set(properties.getTenantIdHeader(),
                principal.getTenantId().toString());
        }
        return execution.execute(request, body);
    }
}

// 注册到 RestTemplate
@Bean
public RestTemplate restTemplate(MultiTenancyProperties properties) {
    RestTemplate restTemplate = new RestTemplate();
    restTemplate.getInterceptors().add(new TenantClientHttpRequestInterceptor(properties));
    return restTemplate;
}
```

**出站（WebClient）**：

```java
@Bean
public WebClient webClient(MultiTenancyProperties properties) {
    return WebClient.builder()
        .filter((request, next) -> {
            TenantPrincipal principal = TenantContext.get();
            if (principal != null) {
                request = request.headers(h ->
                    h.set(properties.getTenantIdHeader(), principal.getTenantId().toString()));
            }
            return next.exchange(request);
        })
        .build();
}
```

**入站**：无需额外处理 —— `TenantIdentityFilter` 已通过 `properties.getTenantIdHeader()` 读取 header（见 §4.2）。

#### 7.2 OpenFeign 传递

Feign 调用本质是 HTTP，通过 `RequestInterceptor` 自动注入 header：

```java
/**
 * Feign 租户传递拦截器。
 * 从当前 TenantContext 提取 tenantId，注入到所有 Feign 请求的 tenant-id-header（由配置决定，默认 X-Tenant-ID）。
 */
public class TenantFeignInterceptor implements RequestInterceptor {

    private final MultiTenancyProperties properties;

    public TenantFeignInterceptor(MultiTenancyProperties properties) {
        this.properties = properties;
    }

    @Override
    public void apply(RequestTemplate template) {
        TenantPrincipal principal = TenantContext.get();
        if (principal != null) {
            template.header(properties.getTenantIdHeader(), principal.getTenantId().toString());
        }
    }
}

// 全局注册（对所有 @FeignClient 生效）
@Configuration
public class FeignConfig {
    @Bean
    public TenantFeignInterceptor tenantFeignInterceptor(MultiTenancyProperties properties) {
        return new TenantFeignInterceptor(properties);
    }
}
```

#### 7.3 gRPC 传递

gRPC 通过 Metadata 传递租户 ID，key 由 `multi-tenancy.tenant-id-header` 配置决定（默认 `X-Tenant-ID`）。

**出站（ClientInterceptor）**：

```java
import io.grpc.*;

public class TenantGrpcClientInterceptor implements ClientInterceptor {

    private final Metadata.Key<String> tenantMetadataKey;

    public TenantGrpcClientInterceptor(MultiTenancyProperties properties) {
        this.tenantMetadataKey = Metadata.Key.of(
            properties.getTenantIdHeader(), Metadata.ASCII_STRING_MARSHALLER);
    }

    @Override
    public <ReqT, RespT> ClientCall<ReqT, RespT> interceptCall(
            MethodDescriptor<ReqT, RespT> method,
            CallOptions callOptions,
            Channel next) {
        return new ForwardingClientCall.SimpleForwardingClientCall<>(
                next.newCall(method, callOptions)) {

            @Override
            public void start(Listener<RespT> responseListener, Metadata headers) {
                TenantPrincipal principal = TenantContext.get();
                if (principal != null) {
                    headers.put(tenantMetadataKey, principal.getTenantId().toString());
                }
                super.start(responseListener, headers);
            }
        };
    }
}
```

**入站（ServerInterceptor）**：

```java
import io.grpc.*;

public class TenantGrpcServerInterceptor implements ServerInterceptor {

    private final Metadata.Key<String> tenantMetadataKey;

    public TenantGrpcServerInterceptor(MultiTenancyProperties properties) {
        this.tenantMetadataKey = Metadata.Key.of(
            properties.getTenantIdHeader(), Metadata.ASCII_STRING_MARSHALLER);
    }

    @Override
    public <ReqT, RespT> ServerCall.Listener<ReqT> interceptCall(
            ServerCall<ReqT, RespT> call,
            Metadata headers,
            ServerCallHandler<ReqT, RespT> next) {

        String headerTenantId = headers.get(tenantMetadataKey);

        // 将 gRPC Metadata 中的 tenant-id-header 写入 Servlet Request attribute，
        // 使下游 TenantIdentityFilter 无感读取并交叉校验
        HttpServletRequest request = GrpcContextKeys.HTTP_REQUEST.get();
        if (request != null && headerTenantId != null) {
            request.setAttribute("grpc.x-tenant-id", headerTenantId);
        }

        return next.startCall(call, headers);
    }
}
```

> gRPC Server Interceptor 通过 `GrpcContextKeys` 获取到桥接的 `HttpServletRequest`，将 gRPC Metadata 中的 `X-Tenant-ID` 填入，使现有的 `TenantIdentityFilter` 无需改造即可完成交叉校验。

#### 7.4 传递规则总结

| 场景 | 生产者（出站） | 载体 | 消费者（入站） |
|------|--------------|------|--------------|
| HTTP 直接调用 | `ClientHttpRequestInterceptor` / `WebClient` filter | `X-Tenant-ID` header | `TenantIdentityFilter` |
| OpenFeign | `TenantFeignInterceptor`（`RequestInterceptor`） | `X-Tenant-ID` header | `TenantIdentityFilter` |
| gRPC | `ClientInterceptor` | gRPC Metadata `X-Tenant-ID` | `ServerInterceptor` → bridge to `HttpServletRequest` → `TenantIdentityFilter` |
| MQ | 生产者写入消息头 | 消息头 `X-Tenant-ID` | 消费者提取后构造 `TenantPrincipal` → `TenantContext.runWithTenant()` |
| 定时任务 | — | — | 任务启动时从配置/JWT 构造 `TenantPrincipal` → 显式调用 `runWithTenant()` |

> **超级管理员**（JWT 无 `tenantCode`）：出站时不注入 `X-Tenant-ID`。下游 `TenantIdentityFilter` 同样识别为超管，跳过租户绑定。
