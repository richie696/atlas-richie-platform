## 策略模块详细设计

### 1. 模块概述

策略模块负责根据租户的隔离模式，在 SQL 执行前动态改写 SQL、切换数据源/Schema 或替换表名。模块通过 `TenancyStrategy` 接口统一行为，并借助 `AbstractTenancyStrategy` 提供公共能力（如连接重置、租户校验），由 `TenancyStrategyFactory` 按模式返回对应实现。

### 2. TenancyStrategy 接口

```java
public interface TenancyStrategy {
    boolean supports(IsolationMode mode);
    void beforeSqlExecute(Invocation invocation, TenantInfo tenantInfo);
}
```

- `supports`：声明该策略支持的隔离模式。
- `beforeSqlExecute`：在 MyBatis-Plus 拦截器执行 SQL 之前调用，允许修改 `Invocation` 或操作连接。

### 3. AbstractTenancyStrategy 抽象基类

```java
public abstract class AbstractTenancyStrategy implements TenancyStrategy {

    protected final MultiTenancyProperties properties;
    protected final TenantInfoProvider tenantInfoProvider;

    public AbstractTenancyStrategy(MultiTenancyProperties properties,
                                   TenantInfoProvider tenantInfoProvider) {
        this.properties = properties;
        this.tenantInfoProvider = tenantInfoProvider;
    }

    /**
     * 确保当前线程存在租户上下文，否则抛异常
     */
    protected void assertTenantPresent() {
        if (TenantContext.getTenantId() == null) {
            throw new BusinessException("Tenant not bound to current context");
        }
    }

    /**
     * 从 Invocation 中获取 Connection，并执行 schema reset（PostgreSQL 模式）
     * 用于 SchemaStrategy 或 DatabaseStrategy 切换后的清理
     */
    protected void resetConnection(Invocation invocation) {
        // 从 invocation 获取 java.sql.Connection
        // 执行 SET LOCAL search_path TO DEFAULT 或类似 reset 语句
        // 仅当数据库支持时执行
    }

    /**
     * 校验租户 ID 格式（Long 类型的 tenantId 无需正则校验，仅做非空判断）
     */
    protected void validateTenantId(Long tenantId) {
        if (tenantId == null || tenantId <= 0) {
            throw new BusinessException("Invalid tenant ID: " + tenantId);
        }
    }
}
```

### 4. TenancyStrategyFactory 工厂

```java
@Component
public class TenancyStrategyFactory {

    private final Map<IsolationMode, TenancyStrategy> strategyMap;

    public TenancyStrategyFactory(List<TenancyStrategy> strategies) {
        // 遍历所有隔离模式，为每个模式匹配对应策略
        this.strategyMap = Arrays.stream(IsolationMode.values())
            .collect(Collectors.toMap(
                Function.identity(),
                mode -> strategies.stream()
                    .filter(s -> s.supports(mode))
                    .findFirst()
                    .orElseThrow(() -> new IllegalArgumentException(
                        "No strategy found for mode: " + mode)),
                (existing, replacement) -> existing,
                () -> new EnumMap<>(IsolationMode.class)
            ));
    }

    public TenancyStrategy getStrategy(IsolationMode mode) {
        TenancyStrategy strategy = strategyMap.get(mode);
        if (strategy == null) {
            throw new IllegalArgumentException("No strategy for mode: " + mode);
        }
        return strategy;
    }
}
```

### 6. TableStrategy 详细实现

```java
public class TableStrategy extends AbstractTenancyStrategy {

    public TableStrategy(MultiTenancyProperties properties,
                         TenantInfoProvider tenantInfoProvider) {
        super(properties, tenantInfoProvider);
    }

    @Override
    public boolean supports(IsolationMode mode) { return mode == IsolationMode.TABLE; }

    @Override
    public void beforeSqlExecute(Invocation invocation, TenantInfo tenantInfo) {
        assertTenantPresent();
        Long tenantId = TenantContext.getTenantId();  // Long 类型
        validateTenantId(tenantId);

        // 表名替换由 DynamicTableNameInnerInterceptor 完成
        // 这里确保表后缀已设置到拦截器的 TableNameHandler
        // 可通过 ThreadLocal 将 tableSuffix 传递给处理器
        TableSuffixHolder.set(tenantInfo.getTableSuffix());
    }
}

// 辅助持有者（ThreadLocal）
public class TableSuffixHolder {
    private static final TransmittableThreadLocal<String> SUFFIX = new TransmittableThreadLocal<>();
    public static void set(String suffix) { SUFFIX.set(suffix); }
    public static String get() { return SUFFIX.get(); }
    public static void clear() { SUFFIX.remove(); }
}
```

对应的 `DynamicTableNameInnerInterceptor` 配置：

```java
@Bean
public DynamicTableNameInnerInterceptor dynamicTableNameInterceptor() {
    DynamicTableNameInnerInterceptor interceptor = new DynamicTableNameInnerInterceptor();
    interceptor.setTableNameHandler((sql, tableName) -> {
        String suffix = TableSuffixHolder.get();
        if (suffix != null && !properties.getIgnoreTables().contains(tableName)) {
            return tableName + suffix;
        }
        return tableName;
    });
    return interceptor;
}
```

### 7. SchemaStrategy 详细实现

```java
public class SchemaStrategy extends AbstractTenancyStrategy {

    public SchemaStrategy(MultiTenancyProperties properties,
                          TenantInfoProvider tenantInfoProvider) {
        super(properties, tenantInfoProvider);
    }

    @Override
    public boolean supports(IsolationMode mode) { return mode == IsolationMode.SCHEMA; }

    @Override
    public void beforeSqlExecute(Invocation invocation, TenantInfo tenantInfo) {
        assertTenantPresent();
        Long tenantId = TenantContext.getTenantId();  // Long 类型
        validateTenantId(tenantId);

        String schemaName = tenantInfo.getSchemaName();
        if (schemaName == null || !schemaName.matches("^[a-zA-Z0-9_]+$")) {
            throw new InvalidSchemaNameException("Schema name validation failed: " + schemaName);
        }
        try {
            Connection conn = invocation.getConnection();

            // schema-auto-create：如果 schema 不存在则自动创建
            if (properties.isSchemaAutoCreate() && !schemaExists(conn, schemaName)) {
                createSchemaWithTables(conn, schemaName);
            }

            // PostgreSQL: SET LOCAL search_path = 'schemaName'
            Statement stmt = conn.createStatement();
            stmt.execute("SET LOCAL search_path TO " + schemaName);
            stmt.close();
            // 标记连接已被修改，归还时需重置（由 resetConnection 完成）
            ConnectionContext.markDirty(conn);
        } catch (SQLException e) {
            throw new SchemaSwitchException("Failed to switch schema to " + schemaName, e);
        }
    }

    /**
     * 检测 schema 是否存在。
     */
    private boolean schemaExists(Connection conn, String schemaName) throws SQLException {
        ResultSet rs = conn.getMetaData().getSchemas();
        while (rs.next()) {
            if (schemaName.equals(rs.getString("TABLE_SCHEM"))) {
                rs.close();
                return true;
            }
        }
        rs.close();
        return false;
    }

    /**
     * 自动创建 schema 并复制表结构。
     * 仅在 schema 不存在时触发（幂等）。
     */
    private void createSchemaWithTables(Connection conn, String schemaName) throws SQLException {
        Statement stmt = conn.createStatement();
        stmt.execute("CREATE SCHEMA IF NOT EXISTS " + schemaName);
        stmt.close();

        // 复制共享 schema 中的业务表到新 schema
        List<String> templates = resolveTableTemplates(conn);
        for (String template : templates) {
            Statement tblStmt = conn.createStatement();
            tblStmt.execute(String.formatted(
                "CREATE TABLE IF NOT EXISTS %s.%s (LIKE public.%s INCLUDING ALL)",
                schemaName, template, template));
            tblStmt.close();
        }
    }
}
```

> **MySQL 兼容性**：`SchemaStrategy` 仅在 PostgreSQL/Oracle 环境下装配。配置校验器在启动时检测 `mode: schema` + MySQL 组合，直接拒绝启动并提示使用 `mode: database`。在 Hybrid 模式下，如果某租户的 `isolation_mode = SCHEMA` 但底层为 MySQL，启动时同样拦截并报错。
>
> **MySQL 中 schema = database 的处理**：MySQL 的 `CREATE SCHEMA` 实质为创建新数据库，无法在单数据库内实现多 schema 隔离。因此 MySQL 用户应直接使用 `mode: database`，而不是 `mode: schema`。组件不提供 MySQL schema 模式的任何支持。

**连接归还重置**：在 `AbstractTenancyStrategy` 的 `resetConnection` 中实现，由拦截器或数据源包装器调用：

```java
protected void resetConnection(Connection conn) {
    try {
        if (ConnectionContext.isDirty(conn)) {
            Statement stmt = conn.createStatement();
            stmt.execute("SET LOCAL search_path TO DEFAULT");
            stmt.close();
            ConnectionContext.clearDirty(conn);
        }
    } catch (SQLException e) {
        log.warn("Failed to reset search_path", e);
    }
}
```

### 8. DatabaseStrategy 详细实现

```java
public class DatabaseStrategy extends AbstractTenancyStrategy {

    private final DynamicTenantDataSource dynamicDataSource;

    public DatabaseStrategy(MultiTenancyProperties properties,
                            TenantInfoProvider tenantInfoProvider,
                            DynamicTenantDataSource dynamicDataSource) {
        super(properties, tenantInfoProvider);
        this.dynamicDataSource = dynamicDataSource;
    }

    @Override
    public boolean supports(IsolationMode mode) { return mode == IsolationMode.DATABASE; }

    @Override
    public void beforeSqlExecute(Invocation invocation, TenantInfo tenantInfo) {
        assertTenantPresent();
        Long tenantId = TenantContext.getTenantId();  // Long 类型
        validateTenantId(tenantId);

        String dataSourceKey = tenantInfo.getDataSourceName();
        if (dataSourceKey == null) {
            throw new IllegalArgumentException("No datasource configured for tenant " + tenantId);
        }

        // 将数据源查找键绑定到线程上下文，供 DynamicTenantDataSource 使用
        DataSourceContextHolder.set(dataSourceKey);
    }
}
```

`DataSourceContextHolder` 使用 `TransmittableThreadLocal`：

```java
public class DataSourceContextHolder {
    private static final TransmittableThreadLocal<String> CONTEXT = new TransmittableThreadLocal<>();
    public static void set(String key) { CONTEXT.set(key); }
    public static String get() { return CONTEXT.get(); }
    public static void clear() { CONTEXT.remove(); }
}
```

`DynamicTenantDataSource`（基于 `AbstractRoutingDataSource`）：

```java
public class DynamicTenantDataSource extends AbstractRoutingDataSource {
    @Override
    protected Object determineCurrentLookupKey() {
        String key = DataSourceContextHolder.get();
        if (key == null) {
            return "shared"; // 默认数据源
        }
        return key;
    }
}
```

### 9. HybridStrategy 详细实现

```java
public class HybridStrategy extends AbstractTenancyStrategy {

    private final TenancyStrategyFactory factory;

    public HybridStrategy(MultiTenancyProperties properties,
                          TenantInfoProvider tenantInfoProvider,
                          TenancyStrategyFactory factory) {
        super(properties, tenantInfoProvider);
        this.factory = factory;
    }

    @Override
    public boolean supports(IsolationMode mode) { return mode == IsolationMode.HYBRID; }

    @Override
    public void beforeSqlExecute(Invocation invocation, TenantInfo tenantInfo) {
        assertTenantPresent();
        validateTenantId(TenantContext.getTenantId());  // Long 类型

        // 按租户的固定隔离模式获取对应策略并委托
        TenancyStrategy delegate = factory.getStrategy(tenantInfo.getMode());
        delegate.beforeSqlExecute(invocation, tenantInfo);
    }
}
```

### 10. 策略与拦截器的协作流程

1. MyBatis-Plus 拦截器（自定义）在 `beforePrepare` 或 `beforeQuery` 中调用 `TenancyStrategyFactory.getStrategy(mode).beforeSqlExecute(invocation, tenantInfo)`。
2. 策略执行预处理：Schema 策略切换 `search_path`；Database 策略设置数据源键；Table 策略设置表后缀。
3. 相应的 MyBatis-Plus 内置拦截器（`TenantLineInnerInterceptor`、`DynamicTableNameInnerInterceptor`）或 `DynamicTenantDataSource` 完成 SQL 改写/路由。
4. SQL 执行后，连接归还前由拦截器或数据源代理调用 `resetConnection` 清理 Schema 变更。

### 11. 关键类图说明

- `TenancyStrategy` ← `AbstractTenancyStrategy` ← `ColumnStrategy`, `TableStrategy`, `SchemaStrategy`, `DatabaseStrategy`, `HybridStrategy`
- `TenancyStrategyFactory` 持有 `Map<IsolationMode, TenancyStrategy>`
- `AbstractTenancyStrategy` 依赖 `MultiTenancyProperties` 和 `TenantInfoProvider`
- `DatabaseStrategy` 依赖 `DynamicTenantDataSource`（通过构造注入）
- `HybridStrategy` 依赖 `TenancyStrategyFactory`（通过构造注入）
