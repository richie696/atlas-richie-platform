# 状态机图解说明

## 🎯 状态机基本概念图解

### 1. 什么是状态？

```
┌─────────────────────────────────────────────────────────────┐
│                        订单状态示例                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐  │
│  │ 待确认  │    │ 已确认  │    │ 制作中  │    │ 已完成  │  │
│  │ PENDING │    │CONFIRMED│    │PREPARING│    │COMPLETED│  │
│  └─────────┘    └─────────┘    └─────────┘    └─────────┘  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**状态**：对象在某一时刻所处的条件或情况。

### 2. 什么是事件？

```
┌─────────────────────────────────────────────────────────────┐
│                        事件示例                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐         │
│  │   CONFIRM   │  │START_PREPARE│  │  COMPLETE   │         │
│  │   确认订单  │  │  开始制作   │  │  完成订单   │         │
│  └─────────────┘  └─────────────┘  └─────────────┘         │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**事件**：触发状态转换的动作或条件。

### 3. 什么是转换？

```
┌─────────────────────────────────────────────────────────────┐
│                        转换示例                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌─────────┐    CONFIRM     ┌─────────┐    START_PREPARE   │
│  │ 待确认  │ ──────────────→ │ 已确认  │ ──────────────────→ │
│  │ PENDING │                 │CONFIRMED│                     │
│  └─────────┘                 └─────────┘                     │
│       │                           │                           │
│       │ CANCEL                    │ CANCEL                    │
│       ▼                           ▼                           ▼
│  ┌─────────┐                 ┌─────────┐                 ┌─────────┐
│  │ 已取消  │                 │ 已取消  │                 │ 待配送  │
│  │CANCELLED│                 │CANCELLED│                 │  READY  │
│  └─────────┘                 └─────────┘                 └─────────┘
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**转换**：从一个状态到另一个状态的过程。

### 状态转换规则说明（含优先级与终态不可变更）

| 当前状态 | 事件 | 目标状态 | 条件 | 动作 |
|---------|------|---------|------|------|
| PENDING | CONFIRM | CONFIRMED | amount > 0 | 设置确认时间 |
| PENDING | CANCEL | CANCELLED | 无 | 设置取消时间 |
| CONFIRMED | START_PREPARE | PREPARING | 无 | 设置制作开始时间 |
| CONFIRMED | CANCEL | CANCELLED | canCancel() | 设置取消时间 |
| PREPARING | READY | READY | 无 | 设置制作完成时间 |
| READY | START_DELIVER | DELIVERING | 无 | 设置配送开始时间 |
| DELIVERING | COMPLETE | COMPLETED | 无 | 设置完成时间 |

说明：
- 引擎支持规则优先级与阈值过滤：`priority` 决定执行顺序，`priorityThreshold` 过滤低优先级规则。
- 当前状态为 `FINAL/ERROR` 时，仅允许带 `attributes.reopen=true` 的“重开”型转换，其余一律拒绝。

## ⚙️ 状态机执行过程图解（静态外观 + 规则引擎参数 + 表达式安全）

### 状态机引擎工作流程

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                             状态机执行流程                                    │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  1. 接收请求                   2. 获取当前状态                   3. 查找转换规则 │
│  ┌─────────────┐              ┌─────────────┐              ┌─────────────┐ │
│  │ 业务代码调用 │ ────────────→ │ 从业务对象或 │ ────────────→ │ 根据当前状态 │ │
│  │ stateMachine│              │ 存储中获取   │              │ 和事件查找   │ │
│  │ Engine.fire()│              │ 当前状态     │              │ 匹配的转换   │ │
│  └─────────────┘              └─────────────┘              └─────────────┘ │
│                                                                             │
│  4. 验证条件                   5. 执行转换                   6. 保存结果     │
│  ┌─────────────┐              ┌─────────────┐              ┌─────────────┐ │
│  │ 检查转换条件 │ ────────────→ │ 更新状态并执 │ ────────────→ │ 将新状态保存 │ │
│  │ 是否满足     │              │ 行转换动作   │              │ 到存储中     │ │
│  └─────────────┘              └─────────────┘              └─────────────┘ │
│                                                                             │
│  7. 返回结果                                                                 │
│  ┌─────────────┐                                                           │
│  │ 返回转换成功 │                                                           │
│  │ 或失败的结果 │                                                           │
│  └─────────────┘                                                           │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 详细执行步骤

```
┌────────────────────────────────────────────────────────────────────────────────────────┐
│                             详细执行步骤                                                │
├────────────────────────────────────────────────────────────────────────────────────────┤
│                                                                                        │
│ 步骤1: 接收请求                                                                         │
│ ┌────────────────────────────────────────────────────────────────────────────────────┐ │
│ │ 业务代码: StateMachine.fire(OrderSm.order, OrderEvent.CONFIRM, orderId, attrs)      │ │
│ │ 参数说明:                                                                           │ │
│ │ - StateMachineType.ORDER: 状态机名称                                                │ │
│ │ - OrderEvent.CONFIRM: 触发事件                                                      │ │
│ │ - order: 业务对象                                                                   │ │
│ └────────────────────────────────────────────────────────────────────────────────────┘ │
│                                                                                        │
│ 步骤2: 获取当前状态                                                                     │
│ ┌────────────────────────────────────────────────────────────────────────────────────┐ │
│ │ 从业务对象获取: order.getStatus() → "PENDING"                                       │ │
│ │ 或从状态机存储获取: stateStorage.getCurrentState()                                   │ │
│ └────────────────────────────────────────────────────────────────────────────────────┘ │
│                                                                                        │
│ 步骤3: 查找转换规则                                                                     │
│ ┌────────────────────────────────────────────────────────────────────────────────────┐ │
│ │ 查找条件: fromState="PENDING" AND event="CONFIRM"                                   │ │
│ │ 找到转换: PENDING → CONFIRMED (CONFIRM)                                             │ │
│ └────────────────────────────────────────────────────────────────────────────────────┘ │
│                                                                                        │
│ 步骤4: 验证条件                                                                         │
│ ┌────────────────────────────────────────────────────────────────────────────────────┐ │
│ │ 条件: "amount > 0 && operatorRole == 'ADMIN'"（SpEL）                               │ │
│ │ 验证: 从 attributes 注入变量（amount/operatorRole）                                 │ │
│ └────────────────────────────────────────────────────────────────────────────────────┘ │
│                                                                                        │
│ 步骤5: 执行转换                                                                         │
│ ┌────────────────────────────────────────────────────────────────────────────────────┐ │
│ │ 更新状态: order.setStatus("CONFIRMED")                                              │ │
│ │ 执行动作: context.attributes['confirmedAt'] = '2025-01-01T00:00:00'        │ │
│ └────────────────────────────────────────────────────────────────────────────────────┘ │
│                                                                                        │
│ 步骤6: 保存结果                                                                         │
│ ┌────────────────────────────────────────────────────────────────────────────────────┐ │
│ │ 保存当前状态: stateStorage.saveCurrentState()                                       │ │
│ │ 保存历史记录: stateStorage.saveStateHistory()                                       │ │
│ └────────────────────────────────────────────────────────────────────────────────────┘ │
│                                                                                        │
│ 步骤7: 返回结果                                                                         │
│ ┌────────────────────────────────────────────────────────────────────────────────────┐ │
│ │ 返回: StateTransitionResult.success(context)                                       │ │
│ │ 包含: 新状态、前一状态、转换信息等                                                    │ │
│ └────────────────────────────────────────────────────────────────────────────────────┘ │
│                                                                                        │
└────────────────────────────────────────────────────────────────────────────────────────┘
```
