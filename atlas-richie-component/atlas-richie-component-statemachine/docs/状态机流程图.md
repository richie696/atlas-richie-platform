## 2. 业务调用层流程图（静态外观 + Redis Stream 异步持久化）

```mermaid
graph TB
    subgraph 业务调用入口
        BizCode[业务代码] --> StaticCall[StateMachine.fire<br/>静态方法调用]
        StaticCall --> GetEngine[获取 StateMachineEngine 实例]
    end

    subgraph 状态转换核心流程
        GetEngine --> ValidateSM[验证状态机是否存在]
        ValidateSM -->|不存在| Error1[返回失败]
        ValidateSM -->|存在| GetCurrentState{获取当前状态}
        GetCurrentState -->|显式传入| UseProvided[使用提供的状态]
        GetCurrentState -->|未传入| ReadStorage[从 StateStorage 读取<br/>Redis 缓存预热机制]
        ReadStorage -->|有值| UseStored[使用存储状态]
        ReadStorage -->|无值| UseInitial[使用初始状态]
        UseProvided --> MatchTransitions[匹配转换规则]
        UseStored --> MatchTransitions
        UseInitial --> MatchTransitions
        MatchTransitions -->|无匹配| Error2[返回失败: 无匹配转换]
        MatchTransitions -->|有匹配| CheckFinal[若当前为 FINAL/ERROR 仅保留 attributes.reopen=true]
        CheckFinal -->|无剩余| Error2
        CheckFinal -->|有剩余| BuildRules[构建 StateTransitionRule]
        BuildRules --> ExecuteRules[执行规则引擎]
        ExecuteRules --> EvalCondition[评估条件 SpEL 表达式<br/>安全检查+慢查询监控]
        EvalCondition -->|条件不满足| Error3[返回失败: 条件不满足]
        EvalCondition -->|条件满足| ExecuteAction[执行动作 SpEL 表达式]
        ExecuteAction --> UpdateContext[更新 StateContext]
        UpdateContext --> SaveState[保存状态到 Redis<br/>StateStorage]
        SaveState --> SaveHistory{是否启用历史}
        SaveHistory -->|是| SaveHist[保存历史记录到 Redis<br/>默认7天过期]
        SaveHistory -->|否| CheckEvent{是否启用事件}
        SaveHist --> CheckEvent
        CheckEvent -->|是| PublishEvent[发布 StateChangedEvent<br/>Spring 事件]
        CheckEvent -->|否| CheckDBEnabled{DB 持久化是否启用}
        PublishEvent --> CheckDBEnabled
        CheckDBEnabled -->|否| Success[返回成功]
        CheckDBEnabled -->|是| PublishStream[发布 StateSyncMessage<br/>到 Redis Stream]
        PublishStream --> Success
    end

    subgraph Redis Stream 异步持久化
        PublishStream --> StreamQueue[Redis Stream<br/>platform:statemachine:db:sync]
        StreamQueue --> StreamConsumer[StateSyncStreamConsumer<br/>多线程并发消费]
        StreamConsumer --> BatchBuffer[ThreadLocal 缓冲区<br/>批量收集消息]
        BatchBuffer --> CheckBatchSize{达到 batchSize?}
        CheckBatchSize -->|否| WaitMore[继续收集]
        CheckBatchSize -->|是| FlushBatch[批量刷写]
        WaitMore --> CheckBatchSize
        FlushBatch --> ReadFromRedis[从 Redis 读取最新状态和历史<br/>StateStorage]

        BatchWrite --> WriteHistory[批量写入<br/>statemachine_state_history]
        WriteCurrent --> ACK[确认消息 ACK]
        WriteHistory --> ACK
        ACK --> End[完成]
    end
```

## 状态转换详细流程（Redis Stream 异步持久化）

```mermaid
sequenceDiagram
    participant Biz as 业务代码
    participant SM as StateMachine<br/>静态门面
    participant Engine as StateMachineEngine
    participant Registry as StateMachineRegistry
    participant Storage as RedisStateStorage
    participant Rule as StateTransitionRule
    participant SpEL as SpEL 表达式引擎
    participant EventPub as ApplicationEventPublisher
    participant Redis as Redis<br/>GlobalCache
    participant Stream as Redis Stream
    participant Consumer as StateSyncStreamConsumer<br/>多线程并发
    participant DB as 数据库<br/>MyBatis Plus

    Biz->>SM: fire(stateMachine, event, businessId, attributes)
    SM->>Engine: fire(...)
    
    Engine->>Registry: getStateMachine(name)
    Registry-->>Engine: StateMachineModel
    
    alt 未显式传入当前状态
        Engine->>Storage: getCurrentState(stateMachine, businessId)
        Storage->>Redis: 查询当前状态
        alt Redis 为空且 DB 启用
            Storage->>Storage: 缓存预热<br/>GlobalCache.getStringCacheWithLock
            Storage->>DB: 从数据库查询
            DB-->>Storage: 当前状态
            Storage->>Redis: 回填缓存
        end
        Redis-->>Storage: currentState (或 null)
        Storage-->>Engine: currentState
        alt 存储中无状态
            Engine->>Engine: 使用初始状态
        end
    end
    
    Engine->>Engine: 构建 StateContext
    Engine->>Engine: 匹配转换规则 transitions
    
    Engine->>Rule: 创建 StateTransitionRule
    Rule->>SpEL: 评估条件表达式<br/>evaluateCondition(context)
    SpEL-->>Rule: 条件结果
    
    alt 条件满足
        Rule->>SpEL: 执行动作表达式<br/>executeAction(context)
        SpEL-->>Rule: 执行结果
        Rule->>Engine: 更新 context.transition
    else 条件不满足
        Rule-->>Engine: 无转换
    end
    
    alt 转换成功
        Engine->>Storage: saveCurrentState(...)
        Storage->>Redis: 保存当前状态<br/>永不过期
        Redis-->>Storage: 成功
        
        alt 启用历史
            Engine->>Storage: saveStateHistory(...)
            Storage->>Redis: 保存历史记录<br/>默认7天过期
        end
        
        alt 启用事件
            Engine->>EventPub: publishEvent(StateChangedEvent)
            Note over EventPub: Spring 事件，供其他监听器使用
        end
        
        alt DB 持久化启用
            Engine->>Stream: 发布 StateSyncMessage<br/>GlobalCache.stream().publish()
            Note over Stream: 消息仅包含同步键<br/>stateMachineName:businessId
        end
        
        Engine-->>SM: StateTransitionResult.success
    else 转换失败
        Engine-->>SM: StateTransitionResult.failure
    end
    
    SM-->>Biz: 返回结果
    
    Note over Stream,Consumer: 异步处理（独立线程池）
    Stream->>Consumer: 批量拉取消息<br/>多线程并发消费
    Consumer->>Consumer: ThreadLocal 缓冲区<br/>批量收集消息
    Consumer->>Consumer: 达到 batchSize 时批量刷写
    Consumer->>Storage: 从 Redis 读取最新状态和历史
    Storage->>Redis: 查询当前状态和历史
    Redis-->>Storage: 最新状态和历史
    Storage-->>Consumer: 状态数据
    Consumer->>DB: 批量写入当前状态表<br/>批量写入历史表
    DB-->>Consumer: 写入成功
    Consumer->>Stream: 确认消息 ACK
```

## 关键设计点说明

1. **Redis 作为单数据源（Single Source of Truth）**：
   - 当前状态和历史记录均先写入 Redis
   - 数据库作为异步持久化副本，通过 Redis Stream 异步写入
   - 缓存预热机制：启动时如果 Redis 为空，自动从数据库加载并回填 Redis

2. **Redis Stream 异步持久化**：
   - 状态变更后直接发布 `StateSyncMessage` 到 Redis Stream（不通过 Spring 事件）
   - 消息仅包含同步键（stateMachineName:businessId），不包含状态数据
   - 消费者从 Redis 读取最新状态和历史，确保数据一致性

3. **多线程并发消费**：
   - `StateSyncStreamConsumer` 支持多线程并发处理（可配置 concurrency）
   - 使用 ThreadLocal 缓冲区批量收集消息
   - 达到 batchSize 时批量刷写到数据库，提升性能

4. **批量写入优化**：
   - 批量查询已存在记录，减少数据库查询次数
   - 批量插入和更新，减少数据库压力
   - 通过唯一约束去重，避免重复写入

5. **历史记录过期**：
   - 默认保留7天（可配置）
   - 自动过期，避免 Redis 数据量过大

## 架构优势

✅ **多实例部署支持**：
- Redis Stream 消费者组自动负载均衡
- 避免重复消费，确保数据一致性
- 支持水平扩展，提升处理能力

✅ **高性能**：
- Redis 作为单数据源，查询延迟低（毫秒级）
- 批量处理减少数据库压力
- 多线程并发消费提升吞吐量

✅ **可靠性**：
- Redis Stream 消息持久化 + ACK 机制
- 缓存预热机制确保数据一致性
- 错误处理策略可配置（SKIP/RETRY）
