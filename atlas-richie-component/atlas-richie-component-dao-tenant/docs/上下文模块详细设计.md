## 上下文模块详细设计

### 1. 模块定位

提供统一的租户 ID 持有与传递机制，对外仅暴露 `TenantContext` 门面，内部根据配置自动在 `ScopedValue` 与 `TransmittableThreadLocal` 之间切换，**无需业务代码或框架显式处理线程间上下文传播**。

### 2. 核心类型

#### 2.1 TenantContextHolder 接口

```java
public interface TenantContextHolder {
    void runWithTenant(String tenantId, Runnable task);
    String getTenantId();
    void clear();
}
```

#### 2.2 ScopedValueHolder

```java
public class ScopedValueHolder implements TenantContextHolder {
    private static final ScopedValue<String> CURRENT_TENANT = ScopedValue.newInstance();

    @Override
    public void runWithTenant(String tenantId, Runnable task) {
        ScopedValue.where(CURRENT_TENANT, tenantId).run(task);
    }

    @Override
    public String getTenantId() {
        return CURRENT_TENANT.isBound() ? CURRENT_TENANT.get() : null;
    }

    @Override
    public void clear() {
        // 自动清除，空实现
    }
}
```

#### 2.3 TransmittableThreadLocalHolder

```java
import com.alibaba.ttl.TransmittableThreadLocal;

public class TransmittableThreadLocalHolder implements TenantContextHolder {
    private static final TransmittableThreadLocal<String> CURRENT_TENANT = new TransmittableThreadLocal<>();

    @Override
    public void runWithTenant(String tenantId, Runnable task) {
        String previous = CURRENT_TENANT.get();
        CURRENT_TENANT.set(tenantId);
        try {
            task.run();
        } finally {
            if (previous != null) {
                CURRENT_TENANT.set(previous);
            } else {
                CURRENT_TENANT.remove();
            }
        }
    }

    @Override
    public String getTenantId() {
        return CURRENT_TENANT.get();
    }

    @Override
    public void clear() {
        CURRENT_TENANT.remove();
    }
}
```

### 3. 门面与初始化

```java
if (properties.isForceThreadLocal() || !isScopedValueAvailable()) {
    holder = new TransmittableThreadLocalHolder();
} else {
    holder = new ScopedValueHolder();
}
```

### 4. 上下文生命周期

**入口绑定**（Servlet Filter）通过 `TenantContext.runWithTenant(tenantId, () -> chain.doFilter())` 完成，无需手动传播。

**线程池使用**：业务线程池只需在创建时通过 TTL 提供的 `TtlExecutors.getTtlExecutor(executor)` 包装，即可自动传递租户 ID，无需再手动调用任何 API。

```java
Executor originalPool = Executors.newFixedThreadPool(10);
Executor ttlPool = TtlExecutors.getTtlExecutor(originalPool);
ttlPool.execute(() -> {
    // TenantContext.getTenantId() 自动可用
});
```

### MQ 场景的上下文传播

- **生产者侧**：发送消息前，业务代码需主动从 `TenantContext.getTenantId()` 获取当前租户 ID，并将其放入消息头（如 `properties.setHeader("tenantId", tenantId)`）。
- **消费者侧**：消费消息时，从消息头提取租户 ID，调用 `TenantContext.runWithTenant(tenantId, () -> { ... })` 绑定后执行业务逻辑。

### 5. 与策略层的交互

所有策略（Column、Table 等）直接调用 `TenantContext.getTenantId()`，对底层实现无感知。在 Table/Schema/Database 策略中需切换租户时，使用 `runWithTenant` 绑定。
